# 📝 填写 USER.md 指南

**目的**: 让智能体更了解你，提供更个性化的服务

---

## 🎯 为什么要填写 USER.md？

USER.md 是智能体对你的认知模型。填写越详细，智能体越能：

- ✅ 理解你的背景和技术水平
- ✅ 使用你熟悉的术语和工具
- ✅ 按照你的偏好格式化输出
- ✅ 避免你不喜欢的表达方式
- ✅ 记住你的习惯和禁忌

---

## 📋 必填字段

### 1. 基本信息

```markdown
**名字**: 张三

**称呼偏好**: 叫我名字就好 / 称呼我为老师 / 随便叫

**职业**: 软件工程师 / 产品经理 / 学生 / 自由职业者
```

**为什么重要**: 智能体需要用合适的称呼和语气与你交流。

---

### 2. 技术背景

```markdown
### 技术栈

- **主要语言**: Python, JavaScript
- **熟悉框架**: React, Django, FastAPI
- **工具**: Git, Docker, VS Code
- **数据库**: PostgreSQL, MongoDB
```

**为什么重要**: 智能体可以根据你的技术栈提供代码示例，避免解释你已知的概念。

---

### 3. 沟通偏好

```markdown
### 输出风格

- [x] 简洁直接（少废话，给答案）
- [ ] 详细解释（包含背景和原理）
- [x] 代码优先（多给示例代码）
- [ ] 图文结合（需要图表说明）

### 技术深度

- [x] 专家级别（跳过基础概念）
- [ ] 中级水平（适当解释）
- [ ] 初学者（需要详细讲解）
```

**为什么重要**: 决定智能体的输出风格和详细程度。

## 🔧 如何填写

1. **复制模板**
   ```bash
   cd /home/admin/projects/opentalon
   cp workspace/USER.md workspace/USER.md.backup
   ```

2. **编辑文件**
   ```bash
   vim workspace/USER.md
   # 或者用你喜欢的编辑器
   ```

3. **删除不适用部分**
   - 删除你不关心的字段
   - 保留对你重要的部分

4. **添加个性化内容**
   - 在"个性化笔记"部分添加特殊偏好
   - 记录你的独特习惯

5. **保存并测试**
   ```bash
   python3 opentalon.py cli
   # 看看智能体是否理解你的偏好
   ```

---

## 💡 提示

- **不必一次填完**: 可以边用边完善
- **随时修改**: 偏好会变，随时更新
- **越具体越好**: "喜欢简洁"不如"回复不超过 10 行"
- **诚实表达**: 不喜欢的就写出来，智能体不会生气
